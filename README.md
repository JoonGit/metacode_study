## 프로젝트 기획안

**프로젝트명 및 한 줄 요약**

- **프로젝트명:** 음성 대화형 RAG 음식 주문 어시스턴트
- **한 줄 요약:** 자연스러운 음성 대화를 통해 메뉴 추천부터 결제까지 지원하는 베리어 프리(Barrier-free) AI 주문 시스템

---

### 1. 타겟 유저 및 해결하려는 문제

이 서비스는 매장 방문객의 주문 접근성을 높이고, 가맹점주의 매장 운영 효율성을 극대화하는 것을 목표로 합니다.

- **매장 방문객 (B2C)**
- **불편함:**
- 메뉴가 너무 많아 선택에 어려움을 겪는 첫 방문자
- 키오스크의 복잡한 UI로 인해 주문이 지연되는 고객 (특히 고령층)
- 시각 장애 등으로 인해 터치스크린 사용 자체가 불가능한 고객

- **가맹점주 (B2B)**
- **불편함:**
- 피크타임(점심/저녁) 시 메뉴 설명 및 고객 응대에 소모되는 인건비와 시간
- 고객 맞춤형 메뉴 추천을 통한 객단가 상승(Up-selling) 수단의 부재

- **💡 해결 방안:**
- 터치스크린 조작 없이 자연스러운 음성 대화만으로 메뉴를 추천받고 장바구니에 담아 결제까지 원스톱으로 진행할 수 있는 환경 제공.

---

### 2. 사용할 데이터셋의 특성

- **매장 정보:** 메뉴 이름, 상세 설명, 맛의 특징, 식재료 정보 등 (RAG Vector DB 구축용)
- **매출 정보:** 시간대별/계절별 판매량, 인기 메뉴, 결제 내역 등 POS 연동 데이터 (MCP를 통한 실시간 DB 쿼리용)

---

### 3. 실제 유저 시나리오

> **상황:** 여름에 매장 메뉴에 익숙하지 않은 첫 방문 고객이 키오스크 앞에서 주문을 시도하는 상황.

- **유저 입력:**
  "오늘 날씨가 더운데 음식 추천해줘"
- **시스템 내부 동작:**

1. **MCP (Model Context Protocol):** LLM이 실시간 날씨 정보를 확인한 후, 가맹점 POS DB에 접근하여 '여름철(6, 7, 8월) 판매량 상위 5개 메뉴' 결제 데이터를 조회.
2. **RAG (Retrieval-Augmented Generation):** 조회된 상위 메뉴(예: 냉면)에 대한 상세한 맛의 특징과 식재료 정보를 Vector DB에서 검색.
3. **추론 및 생성:** 도출된 통계 데이터와 텍스트 정보를 종합하여 자연스러운 추천 문장 생성.

- **시스템 답변:**
  "오늘같이 30도가 넘는 무더운 날에는 우리 매장 여름 판매 1위인 '살얼음 동치미 냉면'을 추천해 드려요. 시원하고 새콤한 육수가 일품인데, 장바구니에 담아드릴까요?"

---

### 4. 핵심 구현 피처 (Core Features)

- **음성 처리 (STT/TTS):** 사용자 음성 명령을 정확하게 텍스트로 변환하고, 생성된 AI의 답변을 자연스러운 음성으로 송출.
- **MCP (Model Context Protocol) 도입:** LLM이 단순 텍스트 생성을 넘어 외부 API(날씨 등) 및 매장 내부 데이터베이스(POS 매출 내역 등)에 직접 접근하고 도구를 사용할 수 있도록 파이프라인 구축.
